# Noon 开发日记规范

## 目的

- 记录当天推进过程
- 保留运行判断、验证路径、风险与下一步
- 与 `91-近期开发节点` 区分开：
  - `开发日记` 保留当天原始过程
  - `近期开发节点` 只保留阶段性变化

## 建议命名

- `YYYY-MM-DD.md`
- 例如：`2026-04-14.md`

## 推荐内容

- 今天推进了什么
- 为什么先做这个
- 跑了哪些验证
- 当前还卡在哪里
- 下一步怎么继续

## 不建议写在这里的内容

- 长期方法论总结
- repo 主文档的正式 green pointer 定义
- 已经确认稳定的阶段结论

这些内容应分别写到：

- `可复用方法论`
- repo 主文档
- `系统架构知识库/91-近期开发节点.md`

## 推荐模板

```md
# 2026-04-14

## 做了什么

- 

## 为什么

- 

## 验证结果

- 

## 风险 / 阻塞

- 

## 下一步

- 
```
