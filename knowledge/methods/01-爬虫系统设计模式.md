## 通用规则

- 多源采集先进入 runtime/stage，再统一进 warehouse
- Web 与运营判断只读 warehouse，不直接读 runtime
- 质量状态不能只看任务是否 completed
- 解析出的业务类型必须和 ETA / 辅助信号分开存

## 当前项目验证过的经验

- 配送方式识别要优先抓 logo / marker，而不是拿 ETA 反推
- 类目链路与关键词链路应共享同一套识别逻辑
- runtime 退化必须结构化暴露，而不是只留日志字符串
