## 推荐模式

- 一个明确的 scheduler / runner 入口
- watchdog 独立检查 stale task、stale worker、report missing、quality degraded
- report 与 watchdog 都要产出结构化摘要
- Web 运行中心直接消费这些摘要，不要求 operator 看日志

## 当前项目提炼出的规则

- `completed` 不等于 `quality full`
- `shared_sync` 需要单独状态机，不能只靠 lock 日志
- 历史缺报要和当前链路故障分开表达
