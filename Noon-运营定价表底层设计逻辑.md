# 📊 Noon 智能定价计算器 (V5 Pro) 底层逻辑与公式拆解

本计算器采用**“参数与计算分离”**的模块化架构，将所有易变的费率规则集中于 `Config` 表，通过“命名管理器（Named Ranges）”与复杂的逻辑嵌套，实现前端 `Pricing` 表的全自动精准计算。

---

## 🏛️ 第一部分：底层架构设计 (Architecture)

### 1. 参数控制中心 (`Config` 表)
`Config` 表是整个计算器的“大脑”，包含四大核心模块：
* **全局变量 (Global Variables):** 通过 Excel 的名称管理器，将汇率、VAT税率、退货上架费等定义为全局绝对引用（如 `UAE_VAT`, `KSA_Exch`），修改一处，全表生效。
* **智能佣金表 (`Comm_Table`):** 将 Noon 官方 60+ 个 SKU 级别的规则拆解为 9 列数据结构：`类目 | UAE计费类型 | UAE基础费率 | UAE阶梯门槛 | UAE超额费率 | KSA规则...`。
* **FBN 配送费表 (`FBN_Table`):** 严格按照重量段上限（0.1, 0.25, 0.5...）进行排序与分档。
* **FBP 配送费表 (`FBP_Table`):** 基于 Drop-off 模式的重量段阶梯费率表。

### 2. 前台计算引擎 (`Pricing` 表)
所有的 SKU 测算在同一行内平行展开，横向划分为 5 大色彩区块，直接引用 `Config` 表的数据，实现 FBN 和 FBP 两种模式的 AB 测试对比。

## 🧮 第二部分：全字段公式拆解 (以 UAE 为例)

> **注：** 以下公式以 `UAE_Pricing` 表的第 2 行 (Row 2) 为例。沙特 (KSA) 逻辑完全相同，仅引用的底层变量名不同。

### 🟢 1. 基础与变量录入区 (A列 - N列)
纯手动录入区域，为后续公式提供基础数据（无公式）。
* **H, I, J列:** 长、宽、高 (cm)
* **K列:** 实重 (kg)
* **L列:** 目标售价 (AED/SAR)
* **M列:** 广告费率 (%)
* **N列:** 退货率 (%)

### ⚪ 2. 费率透视区 (O列 - T列) —— *核心引擎*

#### O列：计费重 (Chargeable Weight)
* **逻辑：** 严格执行 Noon 规则，取“实际重量”与“体积重量（长宽高/5000）”的最大值。
* **公式：**
  ```excel
  =MAX(K2, H2*I2*J2/5000)
  ```

#### P列：VAT Tax (代缴税金)
* **逻辑：** Noon前台售价为含税价，需倒算 VAT（UAE为5%，KSA为15%）。
* **公式：**
  ```excel
  =L2/(1+UAE_VAT)*UAE_VAT
  ```

#### Q列：Comm Rate (综合佣金率%)
* **逻辑：** 解决阶梯累进佣金下显示 "Mixed" 的问题，直接展示实际扣费占售价的比例。
* **公式：**
  ```excel
  =IF(L2>0, R2/L2, 0)
  ```

#### R列：Commission (佣金金额)
* **逻辑：** 根据 `Config` 表定义的 3 种佣金类型（`Fixed` 固定、`Threshold` 门槛跳变、`Sliding` 超额累进）进行智能计算。
* **参数代称：** `V_Type` (类型), `V_Base` (基础费率), `V_Limit` (门槛金额), `V_Tier` (超额费率)
* **公式：**
  ```excel
  =IF(V_Type="Fixed", L2*V_Base, IF(V_Type="Threshold", IF(L2<=V_Limit, L2*V_Base, L2*V_Tier), MIN(L2, V_Limit)*V_Base + MAX(0, L2-V_Limit)*V_Tier))
  ```

#### S列：Comm VAT (佣金税)
* **逻辑：** 平台收取的佣金属于服务费，需额外缴纳增值税。
* **公式：**
  ```excel
  =R2*UAE_VAT
  ```

#### T列：Unit Ret. Fee (单件退货罚款)
* **逻辑：** Noon 2024年新政，因卖家原因退货的罚款取“15 AED/SAR”或“佣金金额的20%”中的较小值。
* **公式：**
  ```excel
  =MIN(15, R2*0.2)
  ```

---

### 🔵 3. FBN 模式损益区 (U列 - AB列)

#### U列：FBN Head Cost (头程 CNY)
* **公式：**
  ```excel
  =(H2*I2*J2/1000000)*E2
  ```

#### V列：FBN Fee (FBN 配送费 AED/SAR)
* **逻辑：** 巨型嵌套 IF 函数。先判断价格 `<25` 或 `>25` 决定收费档位，再通过 `VLOOKUP` 严格匹配计费重所属的尺寸和重量区间。
* **公式结构：**
  ```excel
  =IF(L2<25, IF(O2<=0.1, [0.1kg费率], IF(O2<=0.25, [0.25kg费率]...)), IF(O2<=0.1, [高价0.1kg费率]...))
  ```

#### W列：Ads (广告费)
* **公式：** `=L2*M2`

#### X列：Ret. Admin (退货管理费摊销)
* **公式：** `=T2*N2` (单件罚款 × 退货率)

#### Y列：Log. Loss (物流退货损耗)
* **逻辑：** 发生退货时损失的 FBN 配送费及需支付的上架人工费，乘以退货率作为沉没成本。
* **公式：**
  ```excel
  =(V2 + UAE_Restock)*N2
  ```

#### Z列：Total Cost (FBN 总成本 CNY)
* **逻辑：** `采购 + 头程 + (税+佣金+佣金税+配送费+广告+管理费摊销+物流损耗) * 汇率`
* **公式：**
  ```excel
  =D2 + U2 + (P2 + R2 + S2 + V2 + W2 + X2 + Y2) * UAE_Exch
  ```

#### AA列 & AB列：Net Profit & ROI (净利与投资回报)
* **净利公式：** `=(L2*UAE_Exch) - Z2`
* **ROI公式：** `=IF(Z2>0, AA2/Z2, 0)`

### 🟠 4. FBP 模式损益区 (AC列 - AK列)

#### AC列：FBP Head Cost (头程 CNY)
* **逻辑：** 小包直邮通常不足 0.1kg 按 0.1kg 进位算，加上单件操作费。
* **公式：**
  ```excel
  =(CEILING(O2, 0.1)*F2) + G2
  ```

#### AD列：FBP Fee (FBP 尾程配送费)
* **逻辑：** 阶梯式匹配重量段 (0.25, 0.5, 1.0...) 抓取 FBP Drop-off 费率。
* **公式结构：**
  ```excel
  =IF(O2<=0.25, [0.25kg费率], IF(O2<=0.5, [0.5kg费率]...))
  ```

#### AE列：Reimb. (运费补贴 AED/SAR) —— *正向收入项*
* **逻辑 (UAE)：** `<100` 补 `10`，`>100` 补 `0`。
* **逻辑 (KSA)：** `<100` 补 `12`，`100-500` 补 `6`，`>500` 补 `0`。
* **公式 (以UAE为例)：**
  ```excel
  =IF(L2<100, 10, 0)
  ```

#### AF列 & AG列：Ads & Ret. Admin (广告与罚款摊销)
*(计算逻辑同 FBN 模式)*

#### AH列：Log. Loss (FBP 物流退货损耗)
* **逻辑：** FBP 发生退货时损失 `头程运费(需换算回外币) + 尾程配送费 + 重新上架费`，乘以退货率。
* **公式：**
  ```excel
  =((AC2/UAE_Exch) + AD2 + UAE_Restock) * N2
  ```

#### AI列：Total Cost (FBP 总支出 CNY)
* **逻辑：** 纯支出成本合计（不含补贴抵扣）。
* **公式：**
  ```excel
  =D2 + AC2 + (P2 + R2 + S2 + AD2 + AF2 + AG2 + AH2) * UAE_Exch
  ```

#### AJ列：Net Profit (FBP 净利润 CNY)
* **逻辑：** `(售价 + 运费补贴收入) * 汇率 - 总支出`
* **公式：**
  ```excel
  =(L2 + AE2)*UAE_Exch - AI2
  ```

#### AK列：ROI (投资回报率)
* **公式：**
  ```excel
  =IF(AI2>0, AJ2/AI2, 0)
  ```

## 💡 核心优势总结 (Core Advantages)
1. **绝对防错的阶梯计算 (Bulletproof Tiers):** 放弃简单的 `VLOOKUP` 模糊匹配，改用 `CEILING` 进位和严密的 `IF` 区间嵌套，彻底杜绝计费重在临界点（如 0.251kg）判断错误的致命漏洞。
2. **沉没成本显性化 (Sunk Cost Visibility):** 将极易被财务忽略的 **"佣金VAT"**、**"退货管理罚金"**、**"退货物流损耗"** 全部单列，确保算出的利润是“挤干水分的真净利”。
3. **精准还原账单逻辑 (Statement Accuracy):** FBP 运费补贴未作为成本冲减项，而是作为正向收入加在利润端，完美契合 Noon 官方 Statement 的对账出账逻辑。
